1. BIRTHDAY SYSTEM
════════════════════════════════════════════════════════

DB changes:
ALTER TABLE users ADD COLUMN birth_date TEXT;
ALTER TABLE users ADD COLUMN birthday_coupon_year INTEGER DEFAULT 0;

CREATE TABLE IF NOT EXISTS coupons (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  code TEXT UNIQUE,
  discount_percent INTEGER DEFAULT 20,
  user_id INTEGER,
  expires_at TEXT,
  used INTEGER DEFAULT 0,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

Registration form:
- Add required field "تاريخ ميلادك" (input type="date")
- Save birth_date to DB on register

Birthday check (runs every 24h using setInterval or node-cron):
- Query users where birth_date day+month = today day+month
  AND birthday_coupon_year != current year
- For each matched user:
  1. Generate code: "BDAY-" + userId + "-" + currentYear
  2. Insert into coupons table (20% off, expires 7 days)
  3. Send email via nodemailer to user:
     Subject: "🎂 عيد ميلاد سعيد من MU!"
     Body:
     "كل سنة وأنتِ بخير 🎉
      كود خصم عيد ميلادك: [CODE]
      خصم 20% صالح 7 أيام فقط 🎁"
  4. Update birthday_coupon_year = currentYear for that user

Coupon validation in checkout:
- GET /api/promo/validate?code=XXX
- Check: exists + not used + not expired + belongs to user
- Apply discount, mark used after order placed

4. MULTI-ADMIN MANAGEMENT
════════════════════════════════════════════════════════

ALTER TABLE users ADD COLUMN admin_created_by INTEGER;

ALTER TABLE users ADD COLUMN permissions TEXT DEFAULT '{}';

Admin Panel — new page /admin/admins:

Table columns: Name | Email | Created At | Status | Actions

"Add New Admin" modal fields:
- Full Name
- Email
- Password + "Auto Generate" button (generates "MU@" + 8 random chars)

On Save:
1. Hash password with bcrypt
2. Insert user: role="admin", is_admin=1, permissions=JSON
3. Send welcome email to new admin:
   Subject: "تم إضافتك كمشرف على موقع MU 🔐"
   Body:
   "مرحباً [Name]،

    كلمة المرور: [plain password]
    يرجى تغييرها بعد أول دخول."

Deactivate/Delete admin:
- Cannot delete original admin (admin@mu.com) — protect this
- Deactivate: sets is_admin=0, role="user"
- Delete: removes from DB with confirm dialog

Login update:
- Any user with role="admin" OR is_admin=1 can access /admin
- JWT payload: { userId, role, permissions }
- Each admin section checks its required permission before rendering
